  <action>
Create `tests/measure-bundle.sh` — a bash script that measures the total gzipped size of all clone-authored source files (NOT CDN dependencies).

**What to measure:**
1. All `.js` files under `src/` — these are the clone's JavaScript modules (~23 files across stores, router, proxy, views, widgets, utils)
2. All `.css` files under `src/styles/` — these are the clone's authored CSS files (~12 files)

**What NOT to measure:**
- `index.html` (shell HTML, counted separately if at all)
- CDN scripts (Tailwind CSS, pdf.js, pdf-lib, @noble/hashes, Google Fonts) — these are external dependencies, not the clone's code
- `tests/` directory files
- `.planning/` directory files
- `node_modules/` (doesn't exist — project is CDN-based)

**Script behavior:**
- Use `find src/ -name "*.js" -exec cat {} + | gzip -c | wc -c` for JS measurement
- Use `find src/styles/ -name "*.css" -exec cat {} + | gzip -c | wc -c` for CSS measurement
- Compute raw (uncompressed) totals as well for reference
- Output a single JSON line to stdout: `{"total_raw_bytes":N,"total_gzip_bytes":N,"css_raw_bytes":N,"css_gzip_bytes":N,"js_raw_bytes":N,"js_gzip_bytes":N,"pass":true|false}`
- `pass` is true when `total_gzip_bytes ≤ 180000`
- Use `python3 -c "import json,sys; ..."` if bash JSON formatting gets complex — python3 is guaranteed available
- Script assumes `gzip` is available (standard on Linux)

**Exit codes:** 0 on success (regardless of pass/fail — the JSON reports the verdict). Non-zero only if measurement fails entirely.
</action>

  </verify>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test scripts →localStorage | Test scripts read/write localStorage — must backup/restore user data |
| shell script →filesystem | measure-bundle.sh reads source files — must not modify them |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | tests/test_storage.js →localStorage | mitigate | Backup existing localStorage before tests, restore after. If backup fails (no prior data), clear test keys only. Never leave test data in production localStorage. |
| T-04-02 | Information Disclosure | tests/validation-hub.html | accept | Validation hub is a local development tool, not deployed to production. No secrets are embedded. API tokens are never read or displayed by the hub. |
| T-04-03 | Denial of Service | tests/measure-bundle.sh | accept | Script is read-only (cat + gzip). Cannot modify source files. Worst case: script fails with an error message. |
| T-04-04 | Elevation of Privilege | shell execution | mitigate | measure-bundle.sh uses only `find`, `cat`, `gzip`, `wc`, `python3` — no `sudo`, no `eval`, no `curl`/`wget`. Execute only from project root. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan — all tools use system utilities only. |
</threat_model>
