    go(route): 9-step execution (exact original trace from RECON.md §4.3):
    1. Validate: if (!ROUTES.includes(route)) route = 'dashboard'
    2. Nav toggle: querySelectorAll('[data-route]').forEach(el => el.classList.toggle('active', el.dataset.route === route))
    3. Title lookup: Get [title, subtitle] from titles object
    4. Update #page-title.textContent = titles[route][0]
    5. Update #page-subtitle.textContent = titles[route][1]
    6. Render view: #view.innerHTML = VIEWS[route]()
    7. Scroll to top: window.scrollTo({top:0, behavior:'smooth'})
    8. Close mobile sidebar: toggleSidebar(false)
    9. Post-render hook: if (typeof window['after_' + route] === 'function') window['after_' + route]()

    toggleSidebar(open): Toggles sidebar and backdrop open state.
    - #sidebar.classList.toggle('open', open)
    - #backdrop.classList.toggle('open', open)

    CRITICAL DESIGN DECISIONS (per STATE.md and RESEARCH.md):
    - NO History API (no pushState, no popstate, no hash changes). Browser back button exits the app.
    - NO duplicate-navigation guard. go('dashboard') when already on dashboard fully re-renders.
    - titles object is INSIDE go() — not a separate import. This matches the original pattern.
    - VIEWS functions are pure string generators with NO side effects during render.

    EXPORT: export { ROUTES, VIEWS, go, toggleSidebar }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Route parameter → innerHTML | go() sets #view.innerHTML = VIEWS[route]() — VIEWS functions return trusted template strings, not user input |
| URL/input → go() | Route name validated against ROUTES array — invalid routes default to 'dashboard' |
| User click → toggleSidebar | Toggles CSS classes only — no data exposure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Tampering | go(route) innerHTML injection | accept | VIEWS functions return hardcoded template strings in Phase 02. Phase 03 must ensure user-originated data is sanitized via escapeHTML() before interpolation. |
| T-02-11 | Denial of Service | go() called with invalid route | mitigate | Falls back to 'dashboard' — ROUTES.includes() guards against arbitrary strings. |
| T-02-12 | Elevation of Privilege | window['after_' + route] dynamic hook | accept | Only triggers if a function was explicitly assigned to window. Phase 03 views intentionally set these hooks. No arbitrary code execution risk beyond what the app already allows. |
| T-02-13 | Information Disclosure | go() missing VIEWS key | accept | If VIEWS[route] is undefined, innerHTML becomes 'undefined' (visible string, not a crash). Accept for clone parity — original has same behavior. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package installs — vanilla JS only per D-02. |
</threat_model>
