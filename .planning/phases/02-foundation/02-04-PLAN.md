    FILE 1: src/widgets/toast.js

    toast(msg, icon = '✅'): Shows a toast notification at bottom-center. Single toast slot — new toasts replace old ones.

    Implementation (matches RECON.md lines 237-244):
    - Module-level variable: let _tt = null (timer reference for auto-dismiss)
    - Get #toast element — if null, return early (defensive, matches original's behavior when DOM missing)
    - Set #toast-icon.textContent = icon
    - Set #toast-msg.textContent = msg
    - Remove 'hidden' class from #toast
    - Clear previous timer: clearTimeout(_tt)
    - Set new timer: _tt = setTimeout(() => { t.classList.add('hidden'); }, 3000)

    Edge cases:
    - Concurrent toasts: previous timer is cleared, new toast replaces old (single-slot design)
    - Missing DOM element: getElementById returns null → return early (don't crash)
    - The toast container is styled with Tailwind classes in index.html: fixed bottom-6, centered, glass, shadow-2xl, z-[100]

    EXPORT: export { toast }

    FILE 2: src/widgets/modal.js

    openModal(html): Shows modal overlay and injects HTML content.
    - Get #modal-body element — if null, return early
    - Set #modal-body.innerHTML = html
    - Remove 'hidden' class from #modal-back

    closeModal(): Hides modal overlay.
    - Get #modal-back element — if null, return early
    - Add 'hidden' class to #modal-back

    IMPORTANT: Backdrop-click-to-close is handled by the inline onclick in index.html (Plan 01):
    `<div id="modal-back" class="hidden" onclick="if(event.target===this)closeModal()">`
    The `event.target===this` check ensures clicking the modal body (inner element) does NOT close — only clicking the backdrop background closes.

    No Escape key handler — matches original behavior.

    EXPORT: export { openModal, closeModal }

    DO NOT:
    - Add close buttons to modal content (callers provide their own)
    - Create DOM elements dynamically (use existing #toast, #modal-back, #modal-body from index.html)
    - Use classes or Web Components (factory functions per D-03)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → innerHTML (openModal) | openModal injects HTML directly — callers must sanitize via escapeHTML() |
| User click → clipboard (copyText in Plan 02) | Already handled by clipboard.js; toast is visual feedback only |
| User click → navigation (quickCard onclick) | Route string interpolated into onclick — static values from app code, not user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-14 | Tampering | openModal(html) innerHTML injection | accept | Callers in Phase 03 must use escapeHTML() on user-originated content before passing to openModal. Phase 02 view stubs use hardcoded strings only. |
| T-02-15 | Denial of Service | toast() setTimeout accumulation | mitigate | Single toast slot — clearTimeout cancels previous timer. Maximum 1 active timer at any time. |
| T-02-16 | Spoofing | quickCard onclick route interpolation | accept | Route values are hardcoded strings from app code (not URL params or user input). No injection risk. |
| T-02-17 | Information Disclosure | statCard value display | accept | Values come from localStorage app data — display only, no transmission. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package installs — vanilla JS only per D-02. |
</threat_model>
