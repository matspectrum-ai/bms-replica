**e1Publicar() function (exposed to window)** — exact 5-step pipeline:

Pre-flight:
1. Read s = getSettings()
2. Validate s.cf_token && s.cf_account exist. If not: set publish-log to amber warning "Configure o token Cloudflare nas Configurações antes de publicar." Return.
3. Set projectName = etapa1State.dominio
4. Disable button, set text to "⏳ Publicando..."

Step 1 — Create Project (idempotent):
- POST to `https://api.cloudflare.com/client/v4/accounts/${s.cf_account}/pages/projects`
- Headers: Authorization Bearer token, Content-Type application/json
- Body: { name: projectName, production_branch: 'main' }
- Parse JSON. If !success: check if error code is 8000007, 8000031, or message includes 'exists' — if so, project already exists (continue). Otherwise throw.
- Update log: "📦 Projeto criado/confirmado"

Step 2 — Get JWT Upload Token:
- GET from `https://api.cloudflare.com/client/v4/accounts/${s.cf_account}/pages/projects/${projectName}/upload-token`
- Headers: Authorization Bearer token
- Parse JSON. If !success: throw with error details.
- Extract jwt = d2.result.jwt
- Update log: "🔑 Token de upload obtido"

Step 3 — BLAKE3 Hash (LOCAL computation, no API call):
- Dynamic import: `const { blake3 } = await import('https://esm.sh/@noble/hashes@2.2.0/blake3')`
- Get html = etapa1State.htmlGerado
- Compute base64: `btoa(unescape(encodeURIComponent(html)))` (deprecated pattern preserved for hash compatibility)
- Create toHash: `new TextEncoder().encode(b64 + 'html')`
- Hash: `const hashBytes = blake3(toHash)`
- Convert to hex: `Array.from(hashBytes).map(b => b.toString(16).padStart(2, '0')).join('')`
- Take first 32 chars as fileHash
- Update log: "🔐 Hash calculado"

Step 4 — Upload Asset (uses JWT auth, NOT API token — Pitfall 6):
- POST to `https://api.cloudflare.com/client/v4/pages/assets/upload`
- Headers: Authorization Bearer JWT (NOT API token), Content-Type application/json
- Body: [{ key: fileHash, value: b64, base64: true, metadata: { contentType: 'text/html' } }]
- Parse JSON. If !success: throw.
- Update log: "📤 Arquivo enviado"

Step 5 — Create Deployment (uses API token auth + FormData):
- Create FormData:
  ```javascript
  const fd = new FormData();
  fd.append('manifest', JSON.stringify({ '/index.html': fileHash }));
  ```
- POST to `https://api.cloudflare.com/client/v4/accounts/${s.cf_account}/pages/projects/${projectName}/deployments`
- Headers: Authorization Bearer token (API token, NOT JWT)
- Body: fd (FormData)
- Parse JSON. If !success: throw.
- Update log: "🚀 Deploy criado!"

Success handling:
- Extract url = depJson.result.url || `https://${projectName}.pages.dev`
- Set etapa1State.publicado = { url, projectName, deploymentId: depJson.result.id }
- Update DB: find site by dominio in getDB().sites, set url, deploymentId, status='deploy', atualizado=Date.now(), saveDB()
- toast('🎉 Site publicado!', '🚀')
- setTimeout(() => go('etapa1'), 800) — re-render to show success state

Error handling (all steps in try/catch):
- Set publish-log innerHTML to error message (escapeHTML'd) + recovery tip
- Re-enable button, set text to "🚀 Tentar novamente"
- Do NOT throw — catch at top level only

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → CNPJ API | User-entered CNPJ sent to public BrasilAPI endpoint |
| BrasilAPI response → DOM | JSON response normalized and rendered as company card |
| user input → Cloudflare API | API token + account ID from settings sent to Cloudflare for deploy |
| localStorage → generated site | Company data from localStorage embedded in generated HTML template |
| CDN dynamic import | @noble/hashes loaded from esm.sh at deploy time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Tampering | etapa1.js — CNPJ input validation | mitigate | onlyDigits() strips non-numeric characters. Length check: exactly 14 digits required before fetch. Prevents injection of non-CNPJ strings into API URL. |
| T-03-12 | Tampering | etapa1.js — template XSS in generated site | mitigate | escapeHTML() applied to ALL company data fields (25+ fields) before template literal interpolation. 5 entities escaped. Generated HTML is self-contained — no external user input after generation. |
| T-03-13 | Information Disclosure | etapa1.js — Cloudflare API token in fetch | accept | Token sent via Authorization Bearer header over HTTPS. Encrypted in transit. Token from localStorage (single-user app). No logging of token values. |
| T-03-14 | Spoofing | etapa1.js — esm.sh CDN integrity | mitigate | @noble/hashes version pinned to 2.2.0. esm.sh serves immutable versions. BLAKE3 output verified by Cloudflare on upload — incorrect hash causes Step 4 to fail (safe failure). |
| T-03-15 | Elevation of Privilege | etapa1.js — generated site content | accept | Generated site is static HTML deployed to user's own Cloudflare Pages project. No server-side code execution. No access to admin dashboard data. |
| T-03-16 | Denial of Service | etapa1.js — deploy pipeline retry | accept | Deploy button disabled during execution. Error handling re-enables button. No automatic retry (user-initiated). Timeout handled by fetch's default browser timeout. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No packages installed. @noble/hashes loaded via dynamic import from esm.sh CDN (version-pinned). |
