**Part A: PDF Merge & Download (ETP3-03)**

**baixarPDF() function (exposed to window):**

1. Library check: `if (typeof PDFLib === 'undefined')` → toast error, return
2. Validate pdfState.fileBytes exists (PDF was loaded)
3. Show toast "⏳ Gerando PDF..."
4. Load PDF: `const pdfDoc = await PDFLib.PDFDocument.load(pdfState.fileBytes)`
5. Get pages: `const pages = pdfDoc.getPages()`
6. For each overlay in pdfState.overlays:
   - Get target page: `const page = pages[overlay.page - 1]`
   - Get page dimensions: `const { width, height } = page.getSize()`
   - Calculate scale: original viewport vs actual PDF page dimensions
     - `const scaleX = width / overlay.pageWidth`
     - `const scaleY = height / overlay.pageHeight`
   - **Pitfall 3 compliance — Y-coordinate flip:**
     - pdf.js canvas: Y=0 at top, Y increases downward
     - pdf-lib: Y=0 at bottom, Y increases upward
     - Conversion: `const yPdf = height - (overlay.y * scaleY) - (overlay.size * scaleY * 0.8)`
     - X conversion: `const xPdf = overlay.x * scaleX`
   - Draw text on page:
     ```javascript
     page.drawText(overlay.text, {
       x: xPdf,
       y: yPdf,
       size: overlay.size * scaleY,
       font: await pdfDoc.embedFont(PDFLib.StandardFonts.Helvetica),
       color: PDFLib.rgb(0, 0, 0)
     });
     ```
7. Save PDF bytes: `const pdfBytes = await pdfDoc.save()`
8. Create Blob and trigger download:
   - `const blob = new Blob([pdfBytes], { type: 'application/pdf' })`
   - Create anchor, set href = URL.createObjectURL(blob), download = 'documento-editado.pdf'
   - Click, revoke URL
9. toast('✅ PDF baixado!')

  complemento:   /(?:complemento|comp\.?|apto|apartamento|sala|andar)\s*:?\s*([^,;\n]{2,40})/i,

  municipio:     /(?:Município|Cidade|município|cidade)(?:\s*:\s*|\s+)([^,;\n\-\/]{3,50}?)(?:\s*[,\-\/]|\s*(?:Estado|UF|CEP|$))/i,

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| file upload → pdf.js | User-uploaded PDF parsed by pdf.js library |
| pdf.js canvas → DOM | Rendered canvas content visible in browser |
| overlay text → pdf-lib | User-entered text embedded in downloaded PDF |
| PDF text → DOM | Extracted address text displayed in results card |
| CDN → window globals | pdfjsLib and PDFLib attached to window by CDN scripts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-22 | Tampering | etapa3.js — malicious PDF | accept | PDF processing handled by pdf.js (Mozilla-maintained, sandboxed). JavaScript in PDFs disabled by default. getTextContent() extracts text only, no script execution. File type validated client-side (accept="application/pdf"). |
| T-03-23 | Tampering | etapa3.js — overlay text XSS | mitigate | Overlay text rendered via textContent (not innerHTML) in overlay divs. Downloaded PDF uses drawText — no HTML injection into PDF binary. Address fields displayed with escapeHTML(). |
| T-03-24 | Tampering | etapa3.js — CDN script integrity | accept | pdf.js and pdf-lib loaded from official CDN distribution channels (cdnjs.cloudflare.com, jsdelivr.net). Versions pinned (3.11.174, 1.17.1). Subresource integrity not implemented (original pattern). |
| T-03-25 | Information Disclosure | etapa3.js — PDF content | accept | PDF processed entirely client-side in browser. No PDF data transmitted to any server. Extracted address text stays in DOM (single-user app). Downloaded PDF contains user's own overlays. |
| T-03-26 | Denial of Service | etapa3.js — large PDF | accept | pdf.js renders pages one at a time. Each page allocated as separate canvas. Very large PDFs may consume significant memory but won't crash due to incremental rendering. No server-side processing. |
| T-03-27 | Elevation of Privilege | etapa3.js — file system access | accept | File upload via standard HTML file input — user-initiated, browser-mediated. FileReader API reads file contents. Downloaded PDF via Blob + URL.createObjectURL — standard browser download. No filesystem API usage. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No packages installed. pdf.js and pdf-lib loaded via CDN script tags (version-pinned). |
</threat_model>
