**exportCSV() function implementation:**
1. Read sites = getDB().sites (or [])
2. Define BOM = '\uFEFF' (UTF-8 Byte Order Mark — required for Excel to detect encoding)
3. Define headers array with 10 columns:
   ['Empresa', 'Razão Social', 'CNPJ', 'Domínio', 'URL', 'Tel Empresa', 'Nosso Tel', 'Meta-tag', 'Status', 'Atualizado']
4. Define sep = ';' (semicolon — Brazilian Excel uses comma as decimal separator)
5. Build CSV string: BOM + headers.join(sep) + '\n'
6. For each site, build row array:
   - s.fantasia || ''
   - s.razao || ''
   - fmtCNPJ(s.cnpj)
   - s.dominio || ''
   - s.url || ''
   - s.telefoneEmpresa || ''
   - s.telefoneNosso || ''
   - s.metatag || ''
   - s.status || ''
   - fmtDate(s.atualizado)
7. Escape each cell value: wrap in double quotes, escape internal double quotes as ""
   - `"${String(v).replace(/"/g, '""')}"`
8. Join row with sep, append '\n'
9. Create Blob: `new Blob([csv], { type: 'text/csv;charset=utf-8;' })`
10. Create temporary anchor, trigger download as 'planilha-laboratorio.csv'
11. Revoke object URL after download
12. Show toast('CSV exportado!', '📊')

Use fmtCNPJ and fmtDate from utils/format.js for consistent formatting. The export function is idempotent — can be called multiple times (re-reads from getDB() each time).

Do NOT export the full empresas/sms data — only sites. This matches the original's planilha scope.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → view render | Company and site data read from localStorage injected into HTML via template literals |
| user input → localStorage | Search/filter inputs, status dropdown values, row deletions modifying localStorage |
| view → view | Cross-view state transfer: Banco → Etapa 1 via window._empresaParaEtapa1 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | banco.js — company data XSS | mitigate | escapeHTML() applied to ALL company data fields (razao_social, nome_fantasia, municipio, uf, cnae_descricao) before HTML interpolation. 5 entities escaped per Pitfall 5. |
| T-03-07 | Tampering | planilha.js — site data XSS | mitigate | escapeHTML() on fantasia, razao, dominio, url, metatag fields. fmtCNPJ, fmtDate, formatBRPhone produce safe output (numbers/dates only). |
| T-03-08 | Tampering | planilha.js — CSV injection | mitigate | All CSV cell values wrapped in double quotes with internal quotes escaped as "". No formula injection possible (no = prefix allowed through escaping). UTF-8 BOM ensures correct encoding. |
| T-03-09 | Information Disclosure | banco.js — company data exposure | accept | All companies in localStorage are user's own data. No PII beyond what user intentionally stored. Single-user app — no cross-user data leak possible. |
| T-03-10 | Denial of Service | banco.js — large dataset rendering | accept | Typical dataset sizes (dozens to hundreds of companies) are well within browser rendering limits. No pagination (matches original). If dataset grows very large, browser handles scroll naturally. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No packages installed in this plan. All dependencies already exist from Phase 02. |
</threat_model>
