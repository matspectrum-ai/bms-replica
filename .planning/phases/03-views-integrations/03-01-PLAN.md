<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → view render | User data read from localStorage and injected into HTML via template literals |
| API response → DOM | Cloudflare/SMS24h test responses displayed in DOM |
| File upload → localStorage | JSON backup file parsed and applied to localStorage |
| user input → localStorage | Config form values saved to localStorage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | dashboard.js — localStorage data injection | mitigate | All localStorage data is JSON.parsed with try/catch and schema defaults. If corrupted, returns safe empty arrays. No eval() on stored data. |
| T-03-02 | Tampering | config.js — template literal XSS | mitigate | escapeHTML() applied to all user-provided values (token, account, SMS key) before HTML interpolation. 5 entities escaped: & < > " ' |
| T-03-03 | Tampering | config.js — JSON backup import | mitigate | JSON.parse in try/catch. Structure validation (has db + settings keys). Confirmation prompt before applying. Malformed JSON silently rejected. |
| T-03-04 | Information Disclosure | config.js — API tokens in DOM | accept | Token inputs use type="password" (masked). Tokens stored in localStorage (single-user app — acceptable per ASVS L1 guidance for local-only apps). RefreshHeaderStatus pills don't expose token values. |
| T-03-05 | Elevation of Privilege | config.js — backup import | mitigate | Imported data replaces all localStorage. Confirmation dialog required. User can re-export before import to preserve existing data. No privilege boundary (single user). |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No packages installed. CDN scripts (Tailwind, Google Fonts) from official CDNs already loaded in index.html. Phase 03 does not add new CDN deps (Plan 01 only). |
</threat_model>
